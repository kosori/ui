Kõsori UI uses [Radix UI colors](https://www.radix-ui.com/colors) and is built on top of [Tailwind CSS](https://tailwindcss.com), which means that you can customize the colors by changing the `colors` object in the `tailwind.config.ts` file.

## Structure

Kõsori UI is composed of 6 main colors and their aliases.

| Name                                                                         | Class             |
| ---------------------------------------------------------------------------- | ----------------- |
| <div className='size-3 rounded-full inline-flex bg-primary-solid' /> Primary | `primary-[alias]` |
| <div className='size-3 rounded-full inline-flex bg-grey-solid' /> Grey       | `grey-[alias]`    |
| <div className='size-3 rounded-full inline-flex bg-info-solid' /> Info       | `info-[alias]`    |
| <div className='size-3 rounded-full inline-flex bg-success-solid' /> Success | `success-[alias]` |
| <div className='size-3 rounded-full inline-flex bg-warning-solid' /> Warning | `warning-[alias]` |
| <div className='size-3 rounded-full inline-flex bg-error-solid' /> Error     | `error-[alias]`   |

## Semantic aliases

As Radix UI recommends using semantic color names based on use cases, in Kõsori UI we use the following aliases:

| Step | Class                                | Use Case                                |
| ---- | ------------------------------------ | --------------------------------------- |
| 1    | `[color]-base`                       | App background                          |
| 2    | `[color]-bg-subtle`                  | Subtle background                       |
| 3    | `[color]-bg`                         | UI background                           |
| 4    | `[color]-bg-hover`                   | Hovered UI element background           |
| 5    | `[color]-bg-active`                  | Active / Selected UI element background |
| 6    | `[color]-line`                       | Subtle borders and separators           |
| 7    | `[color]-border`                     | UI element border and focus rings       |
| 8    | `[color]-border-hover \| focus-ring` | Hovered UI element border               |
| 9    | `[color]-solid \| placeholder-text`  | Solid backgrounds                       |
| 10   | `[color]-solid-hover`                | Hovered solid backgrounds               |
| 11   | `[color]-text`                       | Low-contrast text                       |
| 12   | `[color]-text-contrast`              | High-contrast text                      |

Check out the "[Understanding the scale](https://www.radix-ui.com/colors/docs/palette-composition/understanding-the-scale)" documentation of Radix UI for more details.

### Custom colors

Using the "[Create custom palette](https://www.radix-ui.com/colors/custom)" tool, you can create your own custom colors.
You will need to convert the colors to HSL format.

Please read "[Using CSS variables](https://tailwindcss.com/docs/customizing-colors#using-css-variables)" to learn more about how to define your colors as CSS variables.

```css title="globals.css"
:root {
  --grey-1: 300 20% 99%; /* [!code highlight] */
  /* ... */

  --primary-1: 240 20% 99%; /* [!code highlight] */
  /* ... */
}

.dark {
  :root {
    --grey-1: 300 4.2% 9.4%; /* [!code highlight] */
    /* ... */

    --primary-1: 260 8.11% 7.25%; /* [!code highlight] */
    /* ... */
  }
}
```

## Dark mode

You need to set the `darkMode` strategy to `class` in your Tailwind CSS config file.

```ts title="tailwind.config.ts"
import type { Config } from 'tailwindcss';

const config: Config = {
  darkMode: 'class', // [!code highlight]
  // ...
};
```

With this strategy, we can use the `light` class to apply light mode styles and the `dark` class to apply dark mode styles.

```html
<!-- [!code word:ligh | dark] -->
<html class="ligh | dark" />
```

## Using the colors

These are some little examples of how to use the colors in Kõsori UI.

```tsx
<button className='bg-primary-solid text-grey-base hover:bg-primary-solid-hover focus-visible:outline-primary-focus-ring'>
  Hello
</button>
```

```tsx
<hr className='border-t border-grey-line' />
```

```tsx
<input className='border border-grey-border placeholder:text-grey-placeholder-text hover:border-grey-border-hover' />
```
